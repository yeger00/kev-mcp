MCP - Model Control Protocol Server

Welcome to the MCP (Model Control Protocol) server. This service facilitates communication between Claude and Cursor for enhanced AI assistance capabilities and provides access to the CISA Known Exploited Vulnerabilities (KEV) catalog.

Available Endpoints

SSE Endpoint

http://127.0.0.1:8000/sse

This is the main Server-Sent Events (SSE) endpoint that enables real-time communication between Claude and Cursor.

REST API Endpoints

Health Check

GET http://127.0.0.1:8000/rest/health

Check the health status of the server.

Server Status

GET http://127.0.0.1:8000/rest/status

Get current server status and available endpoints.

Check CVE Existence

GET http://127.0.0.1:8000/rest/check-cve?cve_id=CVE-2024-1234

Check if a specific CVE exists in the CISA Known Exploited Vulnerabilities (KEV) catalog and get its details.

Parameters:

  • cve_id (required) - The CVE ID to check (e.g., CVE-2024-1234)

Response includes:

  • Vendor/Project information
  • Product details
  • Vulnerability name and description
  • Date added to KEV catalog
  • Required action and due date
  • Known ransomware campaign use
  • Additional notes and CWEs

Get Recent CVEs

GET http://127.0.0.1:8000/rest/recent-cves?days=7

Get a list of recent CVEs from the CISA KEV catalog.

Parameters:

  • days (optional) - Number of days to look back (1-30, default: 7)

Response includes:

  • Total number of CVEs found
  • Days range searched
  • Full details for each CVE from the KEV catalog

API Documentation

GET http://127.0.0.1:8000/rest/docs

View the OpenAPI documentation for all REST endpoints.

How to Use

  1. Ensure the MCP server is running on port 8000 (default)
  2. Connect to the SSE endpoint at /sse for real-time updates
  3. Use the REST API endpoints at /rest/* to query the CISA KEV catalog
  4. Use Cursor with Claude to leverage the enhanced capabilities provided by this integration

Configuration

The server port can be configured using the MCP_PORT environment variable (default: 8000)
